# 机构遴选页面需求文档

## 一、功能概述

机构遴选用于为项目自动分配评估机构。系统基于机构历史绩效、本次摇号预设比例和已承担项目情况，通过加权随机算法实现公平分配。

**核心功能：**

- 维护评估机构信息及绩效数据
- 维护待分配项目信息
- 按规则自动执行机构遴选
- 跟踪机构年度承担项目情况

---

## 二、页面结构

页面采用「统计概览 + Tab 切换」结构，包含三个模块：

### 2.1 统计概览

位于 Tab 上方，展示全局核心数据一览：

| 统计项 | 数据来源 | 说明 |
| --- | --- | --- |
| 符合摇号资格机构 | 机构列表 | 本次摇号预设比例之和 > 0 且 本年度已承担项目比例 ≤ 本次摇号预设比例的机构数量（已承担比例 = 预设比例时算符合）；遴选时还须满足本次摇号预设比例之和 = 100% |
| 待分配项目/分包 | 项目列表 | 状态为待分配的记录行数（不按项目数量累加） |
| 本次摇号预设比例之和 | 机构列表 | 所有机构的本次摇号预设比例之和（不分页，统计全部机构），以百分比显示 |

**状态标识规则：**
- 符合摇号资格机构：数值 > 0 时正常显示，= 0 时标红
- 待分配项目/分包：数值 > 0 时正常显示，= 0 时置灰
- 本次摇号预设比例之和：= 100% 时显示绿色标识，≠ 100% 时显示红色标识并提示「待调整」

### 2.2 Tab 切换区域

| Tab 标签 | 功能 |
| --- | --- |
| 机构列表 | 维护评估机构基础信息、绩效数据及预设比例 |
| 项目列表 | 维护待分配项目并执行遴选操作 |

### 3.1 功能说明

| 批量删除 | 选中多条记录后批量删除，需二次确认 |
| 查询 | 按机构名称模糊搜索。 |

| 本年度已承担项目比例(%) | 小数 | 系统计算 | 该机构本年度已承担项目数 ÷ 所有机构本年度已承担项目数之和（即所有项目数量字段之和）× 100%，保留 2 位小数；所有机构本年度已承担项目数之和为 0 时，各机构已承担比例均为 0%；删除机构时，该机构已承担项目数从分子中移除并从分母中扣除，按剩余分母重新计算其他机构已承担比例 |

**机构名称标黄规则：** 本次摇号预设比例 = 0% 的机构，其机构名称以黄色字体显示。

**机构名称标绿规则：** 本次摇号预设比例 > 0 且 本年度已承担项目比例 ≤ 本次摇号预设比例的机构（不含已承担比例 = 0% 且预设比例 > 0 的机构），其机构名称以绿色字体显示。

## 五、遴选规则

### 5.1 遴选流程

```
开始
  → 检查前置条件（机构列表、项目列表、本次摇号预设比例之和 = 100%）
  → 资格筛选：筛选符合条件的机构（预设比例 > 0 且 已承担比例 ≤ 预设比例）
  → 计算概率：基于权重系数计算各机构被选中概率
  → 随机分配：为每个项目独立随机分配机构
  → 展示分配结果预览
  → 用户确认
  → 是 → 同步数据到机构列表
  → 结束
```

**步骤详解**

#### 步骤一：资格筛选

> **说明**：资格筛选所用的「本年度已承担项目比例」为本次遴选开始前的持久化值，不受本次待确认的遴选结果影响。

| 场景 | 筛选规则 |
| --- | --- |
| 预设比例 = 0 | 该机构不纳入本次遴选，不具备摇号资格 |
| 预设比例 > 0 | 仅筛选「本年度已承担项目比例」小于等于「本次摇号预设比例」的机构（已承担比例 = 预设比例时算符合） |

#### 步骤二：概率计算

某机构被选中概率 = 该机构权重系数 ÷ 所有符合资格机构权重系数之和。

#### 步骤三：随机分配

- 参与遴选的项目范围：用户选中的待分配记录行；若未选中任何待分配记录，则为所有待分配记录；若用户勾选了已分配记录与待分配记录的混合，则仅取其中待分配记录，已分配记录被忽略（禁止对已分配记录重复执行遴选）；
- 一行记录（项目或分包）作为遴选的最小单位，整体分配给一家机构；
- 为每个参与遴选的项目独立随机分配机构；
- 每个项目仅分配一次（无论项目数量多少）；
- 使用加权随机算法，按步骤 2 计算的概率进行抽取。

#### 步骤四：结果确认与同步

用户确认后执行以下操作：

1. 统计各机构分配到的项目数量（按项目数量字段累加）；
2. 累加到机构列表的"本年度已承担项目数"；
3. 重新计算所有机构的"本年度已承担项目比例"（该机构本年度已承担项目数 ÷ 所有机构本年度已承担项目数之和）。

### 5.2 规则示例

**初始状态：**

机构列表：

| 机构 | 权重系数 | 本次摇号预设比例(%)| 本年度已承担项目数 | 本年度已承担项目比例(%) |
| --- | --- | --- | --- | --- |
| 科工局 | 0.8 | 30% | 0 | 0% |
| 信息中心 | 1.0 | 20% | 0 | 0% |
| 中咨公司 | 1.5 | 50% | 0 | 0% |

项目列表：

| 项目名称 | 项目数量 |
| --- | --- |
| 项目 A | 1 |
| 分包 B | 3 |

**遴选过程：**

1. 资格筛选：首次遴选，预设比例 > 0 的所有机构均符合资格（预设比例 = 0 的机构直接排除）；
2. 概率计算：权重系数之和 = 3.3，科工局 24.24%、信息中心 30.30%、中咨公司 45.45%；
3. 随机分配（示例结果）：
   - 项目 A → 中咨公司
   - 分包 B → 信息中心

**确认后同步：**

机构列表更新：

| 机构 | 本年度已承担项目数 | 本年度已承担项目比例 |
| --- | --- | --- |
| 科工局 | 0 | 0% |
| 信息中心 | 3 | 75% |
| 中咨公司 | 1 | 25% |

**再次遴选（新增项目后）：**

新增项目 C（数量 2），执行遴选：

1. 资格筛选：信息中心（75% ≥ 20%）不符合，科工局和中咨公司符合；
2. 概率计算：权重系数之和 = 2.3，科工局 34.78%、中咨公司 65.22%；
3. 随机分配：项目 C → 科工局（示例）；

确认后：科工局本年度已承担项目数 = 2（33.33%），信息中心 = 3（50%），中咨公司 = 1（16.67%），三机构比例之和约等于 100%（显示时四舍五入，但内部计算始终使用精确比例）。

### 5.3 边界条件与容错

| 场景 | 处理方式 |
| --- | --- |
| 符合资格机构数 = 0 或 所有机构预设比例均为 0 或 本次摇号预设比例之和 ≠ 100% | 提示「当前无符合条件的机构，请调整机构本次摇号预设比例后重试」，禁止执行 |
| 权重系数之和 = 0 | 提示"机构权重系数异常，请检查绩效数据"，禁止执行 |
| 用户取消确认 | 遴选结果不保存，不同步机构列表 |
| 并发操作 | 同一用户多次点击"遴选机构"时，仅首次请求生效，后续请求返回"操作进行中" |
| 同步失败 | 事务回滚，提示"数据同步失败，请重试" |
| 删除机构（有待分配记录） | 删除机构后，按剩余分母重新计算其他机构的本年度已承担项目比例；若该机构已有分配的项目，则随机构一并删除；若该机构有附件，则随机构一并删除 |

### 7.1 数据一致性

- 遴选确认操作必须使用事务，确保机构列表和项目列表同步更新；
- 失败时完整回滚，不允许部分更新。

### 7.2 并发控制

- 遴选操作加全局互斥锁，锁粒度为「机构遴选」业务域级别，防止同一用户重复提交；
- 锁超时时间：30 秒。
- 预览时记录机构数据的版本号/时间戳；用户确认时校验版本，若有变更则提示「机构数据已变更，请重新遴选」并拒绝同步，数据需重新遴选后方可确认。

### 7.3 随机算法

- 使用加权随机算法（Weighted Random Selection）；
- 确保每次遴选结果的随机性和概率准确性；
- 建议使用轮盘赌算法（Roulette Wheel Selection）实现。
